# RegHub v0.3.0.3 Upgrade

1. Do not deploy a red GitHub Actions commit.
2. Replace v0.3.0.2 files with the v0.3.0.3 replace-ready release while preserving `.git`, `.env`, Coolify variables, PostgreSQL, and Keycloak.
3. Commit and push.
4. Wait for Ruff, formatting, compile, Alembic, seed, pytest coverage, pip-audit, and Docker smoke jobs to pass.
5. Redeploy in Coolify only after the workflow is green.
6. Keep Redis Operation Worker OFF unless Redis and a standalone worker are configured and healthy.

For local Windows validation, run:

```powershell
Set-ExecutionPolicy -Scope Process Bypass -Force
.\scripts\Test-RegHubRelease.ps1 -IncludeCoverage
```

The PowerShell script is read-only and retries only recognized transient system failures.
