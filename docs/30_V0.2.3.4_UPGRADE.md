# RegHub v0.2.3.4 Upgrade

1. Create a PostgreSQL backup and record the current healthy v0.2.3.3 deployment.
2. Extract the replace-ready ZIP in a temporary directory.
3. Copy the contents of its `reghub` directory over the repository root and choose Replace.
4. Preserve `.git`, local `.env`, Coolify variables, PostgreSQL, Keycloak and `SESSION_SECRET`.
5. Commit, push and redeploy through the existing Coolify workflow.

```powershell
git status
git add -A
git commit -m "Release RegHub v0.2.3.4 Import Experience Hotfix"
git push origin main
```

No new Alembic revision exists in this release. The normal entrypoint may still run
`alembic upgrade head` safely.

## Live verification

1. Import a new GitHub repository and confirm **View template** appears at completion.
2. Confirm the right-side template card contains thumbnail, title, provider, category and slug.
3. Import the same repository again and confirm status is **Skipped**, not Failed.
4. Click **Continue to update template** and confirm a source-sync operation opens.
5. Confirm the template remains a single database record and its curated publication state is kept.
