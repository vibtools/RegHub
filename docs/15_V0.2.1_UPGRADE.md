# RegHub v0.2.1 Coolify Upgrade

## Before upgrade

1. Create a PostgreSQL backup in Coolify.
2. Record the current healthy v0.2.0 commit/deployment for rollback.
3. Confirm `/api/v1/health` and `/api/v1/ready` return HTTP 200.
4. Preserve all existing Coolify environment values and Keycloak settings.

## Replace and push

Extract the replace-ready ZIP in a temporary directory. Copy the contents of its `reghub` folder
over the existing repository root and choose **Replace**. Do not delete the repository directory;
preserve `.git` and any local `.env`.

```cmd
git status
git add -A
git commit -m "Release RegHub v0.2.1 Provider and Media Stabilization"
git push origin main
```

## Coolify

Redeploy the RegHub application. The existing entrypoint runs:

```bash
alembic upgrade head
python -m scripts.seed
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

The migration adds sync audit columns, backfills missing initial Sync History rows, and creates
`screenshot_jobs`. It does not delete or rename any existing object.

## Verify, one step at a time

1. `/api/v1/health` and `/api/v1/ready` return HTTP 200.
2. `/api/v1/capabilities` returns version `0.2.1`.
3. Open **Providers**; import one repository and confirm its owner appears automatically.
4. Open **Sync History**; old templates have backfill rows and new imports have initial-import rows.
5. Select one template and run **Sync Source**; confirm a new manual sync row appears.
6. Open **Asset Gallery**; add, edit, and delete one HTTPS image.
7. Check `/api/v1/templates/{slug}/assets` and `/freshness`.
8. Check `/api/v1/facets` and `/templates/changes`.

## Screenshot generation

Built-in browser execution is intentionally not included. To generate screenshots, configure the
existing isolated external service variables:

```env
SCREENSHOT_SERVICE_URL=https://your-isolated-service.example/api/screenshot
SCREENSHOT_SERVICE_TOKEN=secret
```

Then set a template Preview URL, select the template, and run **Generate Thumbnail**. Check
**Screenshot Jobs** for success or failure; failed jobs can be selected and retried. v0.2.1 records
jobs but runs the external screenshot request synchronously in the administrator action—there is no
background worker in this release. The external service must enforce DNS and outbound-network rules.

## Rollback

Use Coolify rollback to the previous healthy commit/image. Do not automatically downgrade the
database: v0.2.0 ignores the additive v0.2.1 columns and table.
