# RegHub v0.2.1 API Additions

All pre-existing v1 routes and response formats are preserved.

## Extended template query

```http
GET /api/v1/templates
```

Additive query parameters:

- `tag`
- `language`
- `difficulty`
- `use_case`
- `min_quality` (0–100)
- `updated_since` (ISO-8601)
- `sort`: `featured`, `newest`, `updated`, `quality`, `stars`, `name`
- `order`: `asc`, `desc`

## Change feed

```http
GET /api/v1/templates/changes?updated_since=2026-07-21T00:00:00Z
```

Returns the existing paginated template response ordered by update time.

## Assets

```http
GET /api/v1/templates/{slug}/assets
```

Returns a consistent `{data, meta}` envelope with ordered source, manual, and generated assets.

## Freshness

```http
GET /api/v1/templates/{slug}/freshness
```

Returns source revision, latest sync state, source/update times, analysis time, and stale status.

## Facets

```http
GET /api/v1/facets
```

Returns category, provider, framework, language, and difficulty counts for Published templates.

## HTTP cache validators

Template detail and manifest responses include `ETag` and `Last-Modified`. Clients may send
`If-None-Match`; unchanged resources return HTTP 304.
