# RegHub v0.3.0.3 CI Compatibility and Resilience Hotfix

## Root cause

The v0.3 governance release introduced strict `AdminIdentity` RBAC, immutable audit side effects, and catalog caching. Historical tests still mounted reduced FastAPI/SQLAdmin applications with `SimpleNamespace` identities and containers that did not provide `audit` or `catalog_cache`. Production uses the full `ApplicationContainer`, but the stale fixtures caused 14 deterministic CI failures. Failed tests also stopped coverage at 65.49%, below the unchanged 70% gate.

## Corrections

- Test fixtures now use a real Super Admin `AdminIdentity`.
- Reduced admin test applications install the same identity middleware expected by governed views.
- Catalog cache reads/writes bypass safely when no cache exists and fail open on transient backend errors.
- Audit and cache invalidation failures after operation completion are logged as warnings and never rewrite the persisted operation outcome.
- The 70% coverage threshold remains unchanged.
- Retry wrappers are limited to transient environment/process commands; source, Git, and database data are never edited.

## Compatibility

No route, response field, migration, role, feature, token, template, runtime setting, or Coolify boundary is removed or renamed.
