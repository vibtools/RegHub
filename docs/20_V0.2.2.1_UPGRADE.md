# RegHub v0.2.2.1 Hotfix Upgrade

## Before replacement

1. Record the current healthy v0.2.2 deployment/commit.
2. Keep PostgreSQL, Keycloak, Coolify variables, `.git`, `.env`, and `SESSION_SECRET` unchanged.
3. A database backup is recommended even though this hotfix adds no migration.

## Replace and deploy

Extract the replace-ready archive in a temporary folder. Copy the contents of its `reghub` folder
over the existing repository root and choose **Replace**.

```cmd
git status
git add -A
git commit -m "Release RegHub v0.2.2.1 UI and Operations Hotfix"
git push origin main
```

Redeploy RegHub in Coolify. The existing entrypoint may run `alembic upgrade head`; it is a no-op
because the Alembic head remains `20260721_0004_operations_runtime_settings`.

## First browser verification

1. Open `/admin/settings`.
2. Confirm Feature controls, Integrations, and Add custom API appear as separate tabs.
3. Open Integrations and expand each API editor one at a time.
4. Open `/admin/github-import` and start one import.
5. Confirm the operation page shows progress, logs, and `Live stream` or `Polling fallback`.
6. Open `/admin/operations` and `/admin/asset-gallery`.
7. Repeat Settings and operation detail at a narrow/mobile browser width.

## Rollback

Rollback to the previous v0.2.2 commit or Coolify deployment. No database downgrade is needed.
