# RegHub v0.3.1.0 Audit Summary

## Scope compliance

- Feature additions: none.
- API endpoint additions/removals: none.
- New services or repositories: none.
- New providers/integrations/admin pages/Settings: none.
- Docker, GitHub Actions, Coolify or developer-workflow changes: none.
- Database tables/columns removed: none.

## Findings resolved

1. Legacy roleless cookies could be interpreted too permissively by compatibility behavior. They are
   now rejected and require a fresh verified OIDC identity.
2. Browser session boundaries reused common root key material directly. Each boundary now receives a
   purpose-derived key without deployment changes.
3. Logout previously cleared only part of the local session state. All local cookies and the existing
   identity-provider session path are now handled.
4. AI metadata enrichment did not enforce private classification at every boundary. Adapters preserve
   visibility and the AI client independently refuses private repositories.
5. Registry analysis generated deployment-oriented intelligence that belongs to YGIT. Generation and
   persisted obsolete values are removed while registry metadata remains.
6. Several model/database lengths, constraints and indexes had drifted. The additive migration aligns
   them, removes exact duplicate assets and preserves all authoritative records.
7. Historical generated reports and local build/test artifacts created repository noise. The current
   release package excludes them.

## Residual deployment checks

Real Keycloak logout behavior, private-provider credentials, PostgreSQL production data shape, Coolify
proxy behavior and the complete locked-dependency CI suite remain deployment/CI checks. Production
promotion must wait for the unchanged GitHub Actions workflow to pass.
