# v0.3.1.2 Forensic Release Hardening

This hotfix closes release-engineering and production-safety gaps discovered while validating
v0.3.1.1 on Windows and GitHub Actions.

It does not redesign RegHub or move existing subsystems. The architecture findings concerning
operations, AI enrichment, screenshot orchestration, API credential authority and runtime settings
remain documented work for a separately planned release because removing them in a hotfix would
break existing contracts.

## Security

Production configuration now fails closed when the development session secret is used, when the
public or OIDC issuer URL is not HTTPS, or when all peers are trusted as reverse proxies.

## Startup

Container startup acquires a PostgreSQL advisory lock before running the existing Alembic upgrade
and idempotent seed operations. This prevents multiple replicas from racing the migration chain.

## CI and packaging

The project continues to install as a normal wheel. Generated `build/` and `dist/` output is removed
before repository-integrity tests. CI checks dependency consistency, verifies exactly one Alembic
head, runs the complete coverage suite, exports exact installed third-party versions, excludes only
the private RegHub package, and runs strict `pip-audit` without dependency re-resolution.
