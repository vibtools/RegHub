# Upgrade to RegHub v0.3.1.2

No database migration is added.

Production must provide:

- a non-default `SESSION_SECRET` of at least 32 characters;
- `SESSION_COOKIE_SECURE=true`;
- an HTTPS `PUBLIC_BASE_URL`;
- an HTTPS `OIDC_ISSUER_URL`;
- explicit trusted proxy CIDRs; `*` is rejected.

The existing Docker/Coolify deployment flow remains. Startup still performs Alembic upgrade and
seed automatically, but concurrent containers now serialize that step with a PostgreSQL advisory
lock.
