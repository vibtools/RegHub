# RegHub v0.2.3.2 Hotfix

## Settings

All Settings forms submit through `fetch()` and replace only the Settings content shell. The active section is stored in the URL, session storage, form return target, and server-rendered active classes. If JavaScript is unavailable or a browser performs a normal form submission, the response still renders the originating section.

## Original repository endpoint

```http
GET /api/v1/templates/{slug}/repository
```

The endpoint returns the canonical source repository URL, adapter, default branch, external repository ID, and latest successful source revision. It is protected by the existing Catalog API feature and `catalog` service-token scope.

## Operation logs

Completed operations open at the first log line. Running operations continue following the newest line. Start and Latest controls are available, structured log context is formatted, and terminal rows use forced compact sizing.

## UI cleanup

Import pages retain actionable availability, success, and error states but no longer display internal implementation commentary, credential-mode banners, or Operations Console explanations.
