# Upgrade to RegHub v0.3.2.0

## Required production configuration

Before deployment, set independent random values of at least 32 characters for:

```text
SESSION_SECRET
RUNTIME_ENCRYPTION_KEY
AUDIT_SIGNING_KEY
```

Do not reuse a value between these variables. Retain the previous runtime and audit keys in
`RUNTIME_ENCRYPTION_PREVIOUS_KEYS` and `AUDIT_SIGNING_PREVIOUS_KEYS` during key rotation.

`PUBLIC_BASE_URL` must be the HTTPS origin `https://reghub.ygit.dev`, and its hostname must be listed
in `ALLOWED_HOSTS`. OIDC and any configured AI/screenshot endpoints must use HTTPS. Production
`DATABASE_URL` must use `postgresql+asyncpg://`.

## Deployment sequence

1. Back up PostgreSQL and existing environment variables.
2. Update the required keys and HTTPS/host configuration.
3. Deploy the v0.3.2.0 source through the existing Coolify Git workflow.
4. Startup obtains the PostgreSQL advisory lock, applies the existing migration head, and seeds.
5. Verify `/api/v1/health`, `/api/v1/ready`, OIDC login, registry catalog, imports and publication.

No new migration is introduced by v0.3.2.0.
