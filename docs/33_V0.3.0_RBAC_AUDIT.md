# RegHub v0.3.0 RBAC and Immutable Audit

## Keycloak roles

The claim path is configured with `OIDC_ROLE_CLAIM`. The default external-to-internal mapping is:

```json
{
  "super_admin": ["reghub-admin", "reghub-super-admin"],
  "security_admin": ["reghub-security-admin"],
  "publisher": ["reghub-publisher"],
  "editor": ["reghub-editor"],
  "viewer": ["reghub-viewer"]
}
```

Legacy `reghub-admin` remains a Super Admin. Existing signed admin cookies without a role field are
also treated as legacy Super Admin cookies until they expire.

## Permission matrix

| Role | Registry read | Edit/import/sync/media | Publish | Settings/API security | Audit/operation administration |
|---|---:|---:|---:|---:|---:|
| Viewer | Yes | No | No | No | No |
| Editor | Yes | Yes | No | No | Own operation retry/cancel |
| Publisher | Yes | Yes | Yes | No | Own operation retry/cancel |
| Security Admin | Yes | No | No | Yes | Yes |
| Super Admin | Yes | Yes | Yes | Yes | Yes |

Retry authorization is based on the original operation type, preventing a lower-privileged user from
replaying a publication or security-sensitive operation. A non-security administrator may cancel
only an operation requested by the same identity.

## Immutable audit chain

Audit events include actor, roles, action, resource, outcome, request ID, client IP, bounded details,
previous hash, signing-key ID and event hash. Sensitive nested values are redacted before storage.

Every event HMAC covers the normalized values exactly as stored and the previous event hash. Direct
row modification, deletion or reordering breaks verification. Operation history may be cleared for
UX/retention purposes, but the corresponding audit event remains.

Audit events are read-only in SQLAdmin. Chain status is shown at `/admin/governance`.
