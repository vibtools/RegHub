# RegHub v0.2.3.1 Hotfix

## Settings tab persistence

Every Settings form records its originating tab before submission. The server also infers a safe
fallback tab from the submitted action. Success and error responses reopen the same workspace:
Feature controls, Integrations, API Manage, or Add custom API.

## API verification repair

The v0.2.3 API checker accidentally sent requests to the mounted SQLAdmin Starlette application.
That sub-application does not own `/api/v1/*`, and its error template attempted to resolve the
`admin:statics` route in the wrong routing context. v0.2.3.1 sends checks through the root FastAPI
application with an `httpx.ASGITransport`, a short-lived internal token, no redirects, no cache, and
structured per-route results.

## API endpoint registry

API Manage lists every supported public endpoint. Each row provides:

- HTTP method, path, purpose, and required service-token scope
- **Check** — verifies only that endpoint without reloading the page
- **Use** — copies the full URL, token header, PowerShell example, curl example, and permission scope

Dynamic template routes use the most recently updated published template. When no published template
exists, those checks are safely marked unavailable instead of returning a misleading failure.

## Terminal diagnostics

Operation logs now include bounded and redacted diagnostic context. The UI uses a compact one-row
terminal layout. Structured diagnostic JSON is appended inline instead of occupying an empty second
CSS grid row.
