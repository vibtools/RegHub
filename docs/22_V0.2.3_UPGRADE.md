# RegHub v0.2.3 Coolify Upgrade

## Compatibility rules

- Preserve `https://reghub.ygit.dev`, PostgreSQL, Keycloak, and current Coolify variables.
- Keep `SESSION_SECRET` unchanged.
- Preserve `.git` and any local `.env` during replacement.
- Do not delete or recreate the database.
- RegHub remains registry-only; YGIT remains the deployment engine.

## Before deployment

1. Create a PostgreSQL backup in Coolify.
2. Record the current healthy v0.2.2.1 commit/deployment.
3. Verify `/api/v1/health` and `/api/v1/ready`.

## Replace and deploy

Extract the replace-ready archive. Copy the contents of its `reghub` directory over the existing
repository root and choose **Replace**.

```cmd
git status
git add -A
git commit -m "Release RegHub v0.2.3 Operations and API Access"
git push origin main
```

Redeploy in Coolify. The unchanged entrypoint runs:

```bash
alembic upgrade head
python -m scripts.seed
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Migration `20260721_0005` creates only API policy, token, and block-rule tables.

## First verification

1. Open `/admin/settings#api-manage-pane`.
2. Keep **Development Mode** while testing.
3. Create one scoped service token and copy it immediately.
4. Run **Check API endpoints**.
5. Import a repository that already exists; confirm **Skipped / No change**, not Failed.
6. Open `/admin/operations`; verify detailed compact logs and Clear History.
7. Open `/admin/asset-gallery`; verify template and asset search.
8. Verify `/api/v1/templates?tag=astro` remains healthy.

## Enabling Live Mode

Do not enable Live Mode until the YGIT consumer has a tested token. Send the token server-to-server:

```http
Authorization: Bearer vt_reg_xxxxx
```

After YGIT succeeds with the token, enable Live Mode in Settings. No Coolify redeploy is required.

## Rollback

Rollback to the prior v0.2.2.1 commit/deployment. Do not automatically downgrade PostgreSQL;
v0.2.2.1 ignores the additive v0.2.3 tables. Investigate logs before any controlled schema
rollback.
