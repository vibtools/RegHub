# Upgrade to RegHub v0.3.1.1

The deployment workflow is unchanged.

1. Take the normal PostgreSQL backup.
2. Replace the project files while preserving `.git` and the local `.env`.
3. Run the existing `git add -A`, commit and push workflow.
4. Allow the existing GitHub Actions workflow to complete.
5. Redeploy through Coolify exactly as before.

The existing entrypoint automatically runs:

```text
alembic upgrade head
python -m scripts.seed
uvicorn ...
```

Migration `20260722_0008_final_stabilization_hotfix` is data-only, deterministic and automatic.
It adds no table or column and requires no manual SQL or cleanup.

## Post-deployment verification

```text
/api/v1/health
/api/v1/ready
/api/v1/capabilities
/admin
/admin/settings
/admin/operations
/admin/governance
```

Confirm one published template detail/manifest request, one repository import or source sync, assets,
OIDC login/logout and publication state. No Redis, Docker, GitHub Actions or Coolify configuration
change is required.
