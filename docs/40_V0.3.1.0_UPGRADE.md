# Upgrade to RegHub v0.3.1.0

The deployment workflow is unchanged.

1. Take the normal PostgreSQL backup.
2. Replace the project files while preserving `.git` and any local `.env`.
3. Run the existing `git add -A`, commit and push workflow.
4. Allow the existing GitHub Actions workflow to complete.
5. Redeploy through Coolify exactly as before.

The existing entrypoint automatically runs:

```text
alembic upgrade head
python -m scripts.seed
uvicorn ...
```

No new environment variable, service, command, worker or manual migration step is introduced.

## Expected administrator session behavior

Existing roleless/versionless administrator cookies are intentionally rejected. An administrator may
be redirected through the normal Keycloak login once after deployment. Existing Keycloak role values
and mappings remain valid.

## Post-deployment verification

Use the same checks already required for the current release:

```text
/api/v1/health
/api/v1/ready
/api/v1/capabilities
/admin
/admin/settings
/admin/operations
/admin/governance
```

Confirm login/logout, one public template detail/manifest request, and one safe source synchronization.
No Redis, Docker, GitHub Actions or Coolify configuration change is required.
