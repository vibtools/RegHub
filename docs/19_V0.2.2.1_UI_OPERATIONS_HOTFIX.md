# RegHub v0.2.2.1 UI & Operations Hotfix

## Deployment

Replace the v0.2.2 source with the v0.2.2.1 replace-ready project, preserve `.git`, `.env`, Coolify
variables, PostgreSQL, Keycloak, and `SESSION_SECRET`, then run the normal Coolify redeploy.

No Alembic revision was added. `alembic upgrade head` remains safe and stays at
`20260721_0004_operations_runtime_settings`.

## Browser checks

1. Open `/admin/settings` and confirm the page has three compact tabs.
2. Open Integrations and expand GitHub, GitLab, Bitbucket, AI, and Screenshot one at a time.
3. Start one GitHub import and confirm the operation page immediately shows queued/running status.
4. Confirm debug logs appear through SSE or the visible “Polling fallback” mode.
5. Open `/admin/operations`, `/admin/asset-gallery`, and all import pages at desktop and mobile width.
6. Confirm no page forms a narrow vertical strip or an unbounded horizontal/vertical layout.

## Operation update transport

The browser first attempts authenticated Server-Sent Events. If the reverse proxy or browser
interrupts SSE twice, the page automatically switches to authenticated JSON polling every 1.5
seconds. Existing server-rendered logs remain visible even when JavaScript or live transport fails.
