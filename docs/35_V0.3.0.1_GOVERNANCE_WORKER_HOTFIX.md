# RegHub v0.3.0.1 Governance UI and Redis Worker Hotfix

## Governance UI root cause

The v0.3.0 template extended the shared custom-page layout but overrode SQLAdmin's outer `content`
block. That bypassed the mandatory full-width Bootstrap column and allowed the SQLAdmin row to shrink
and overflow. The hotfix uses `reghub_content` and keeps all content inside the shared responsive shell.

## Redis worker runtime switch

`redis_worker` is available under Settings → Feature controls → Operations.

- **OFF:** new operations execute in the existing web-process runner.
- **ON:** new operations enter the Redis durable queue.
- Activation is rejected unless `REDIS_URL` is reachable and a standalone worker heartbeat exists.
- Turning the switch OFF does not cancel work. New operations return to in-process execution while the
  worker drains any operations that were already queued.

Provisioning Redis and the standalone worker remains an infrastructure task. Once provisioned in
standby mode, normal activation/deactivation is handled from Settings without replacing the web app.

## Compatibility

No public route, response field, role, operation state, database record, or migration was removed.
