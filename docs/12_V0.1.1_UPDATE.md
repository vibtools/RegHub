# RegHub v0.1.1 — Replace-and-Redeploy Guide

## Scope

This is a drop-in production patch based on v0.1.0. It keeps the existing database,
Keycloak login, domain, public API, SQLAdmin CRUD, lifecycle actions, and Coolify setup.

## Files and configuration

Replace the v0.1.0 source files with the contents of the v0.1.1 ZIP. Do not delete your
local `.env` file. Coolify environment variables are stored outside the Git repository and
remain unchanged.

Set the optional GitHub token in Coolify:

```env
GITHUB_TOKEN=github_pat_...
GITHUB_TIMEOUT_SECONDS=15
GITHUB_ALLOW_PRIVATE_REPOSITORIES=false
```

Recommended token ownership:

- Create a fine-grained PAT from a company-managed GitHub service user.
- Make that service user a member of the `vibtools` organization when organization
  repositories are required.
- Grant only Metadata read and Contents read to the required repositories.
- Store the token as a Coolify runtime secret; Build Variable must remain disabled.
- Do not commit or paste the token into source files, screenshots, logs, or chat.

Public repository imports continue to work without a token, but GitHub applies a lower
unauthenticated rate limit.

## Git commands

```bash
git status
git add .
git commit -m "Release RegHub v0.1.1"
git push origin main
```

Redeploy the application in Coolify after the push. The existing entrypoint runs:

```bash
alembic upgrade head
python -m scripts.seed
```

v0.1.1 adds no new migration, so this is idempotent and preserves existing records.

## Verification

1. `GET https://reghub.ygit.dev/api/v1/health` returns healthy.
2. `GET https://reghub.ygit.dev/api/v1/ready` confirms database readiness.
3. `/admin/template/list` opens without HTTP 500.
4. Status, Featured, and Framework filters render and work.
5. `/admin/github-import` displays authenticated mode when `GITHUB_TOKEN` is set.
6. Import an Astro repository containing `astro.config.*` or an `astro` package dependency.
7. The imported draft is assigned to the Astro framework.
8. Existing Template API responses and published templates remain unchanged.

## Rollback

Use Coolify's previous successful deployment, or revert the v0.1.1 Git commit and redeploy.
No database rollback is required because there is no database migration in this release.
