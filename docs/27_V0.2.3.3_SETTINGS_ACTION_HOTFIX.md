# RegHub v0.2.3.3 Settings Action Hotfix

## Problem

The live v0.2.3.2 deployment returned HTTP 404 for Settings mutations submitted through the generic
page URL. The client then kept the form in a busy state, which made the interface appear frozen.

## Resolution

Settings mutations use the dedicated `POST /admin/settings/action` route. The original
`POST /admin/settings` behavior remains available for browsers without JavaScript. After a
successful mutation, only the active tab pane and its server feedback are refreshed. The navigation,
other tabs, and page shell are not replaced.

The browser aborts a request after 45 seconds, restores the submitted button, and displays an inline
error without reloading the page.
