# RegHub v0.3.2.0 Validation Contract

Promotion to `main` requires both GitHub Actions jobs to pass.

## quality-and-integration

- pinned GitHub action revisions and read-only repository permissions;
- package installation and `pip check`;
- Ruff lint/format and Python compilation;
- static security, release-tree, release-file manifest and production-config verifiers;
- one Alembic head, PostgreSQL upgrade and seed;
- full pytest coverage of at least 70%;
- exact installed third-party snapshot and strict `pip-audit`.

## docker-smoke

- wheel-based multi-stage image build;
- declared non-root `reghub` user;
- read-only filesystem, temporary `/tmp`, no Linux capabilities, no-new-privileges;
- serialized migration/seed startup;
- database-aware `/api/v1/ready` success;
- unconditional container log capture and cleanup.

Environment-specific Coolify, Keycloak, provider credentials, DNS, TLS and backup restoration remain
post-deployment operational checks.
