# RegHub v0.3.0 CI and Release Gate

GitHub Actions now runs two production gates.

## Quality and integration

- Python 3.13 dependency installation
- Ruff lint and formatting
- Python compilation
- Real PostgreSQL 17 and Redis 7 integration services
- Alembic upgrade to head
- idempotent seed
- test suite with 70% application coverage floor
- `pip-audit` dependency vulnerability check

## Docker smoke

- multi-stage Docker image build
- container start against PostgreSQL
- entrypoint migration and seed
- HTTP health verification
- failure log capture

A release should not be promoted when either gate fails. Deployment-time checks still include
Keycloak, Coolify proxy networks, live provider credentials, worker heartbeat and backup/restore.
