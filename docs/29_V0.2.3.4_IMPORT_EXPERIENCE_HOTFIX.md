# RegHub v0.2.3.4 Import Experience Hotfix

## Successful import

After a successful import, the operation page receives a template summary through the normal status
or SSE response and shows:

- View Template beside Refresh
- Thumbnail or a safe placeholder
- Name, slug, short description, status
- Provider, category, framework and framework version
- Quality score and repository adapter
- Template details and original repository links

The card appears without a page reload when an active operation finishes.

## Existing repository

A repository that already exists in RegHub completes as `skipped`, with 100% progress and no error
message. The page displays **Already found** and links the existing template. Import History remains a
successful no-change record.

**Continue to update template** creates a new single-template `sync_templates` operation. It refreshes
source metadata, analysis, media, manifest and history through the established synchronization
service while preserving curated fields and publication state.

## Compatibility

No route or model was removed. No database migration is required. Operations, Settings, API access,
service tokens, assets, screenshots, Keycloak and Coolify behavior remain compatible with v0.2.3.3.
