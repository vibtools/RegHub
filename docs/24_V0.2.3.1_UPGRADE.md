# RegHub v0.2.3.1 Upgrade

## Compatibility

This is a source-only hotfix. There is no Alembic migration and no data transformation. Preserve the
current PostgreSQL database, `.git`, local `.env`, Keycloak configuration, Coolify variables,
`SESSION_SECRET`, runtime integration secrets, service tokens, and API block rules.

## Deploy

1. Take a PostgreSQL backup and record the current healthy v0.2.3 commit.
2. Extract the replace-ready ZIP in a temporary folder.
3. Copy the contents of `reghub/` over the existing repository root and choose **Replace**.
4. Commit and push:

```cmd
git status
git add -A
git commit -m "Release RegHub v0.2.3.1 API and Operations Hotfix"
git push origin main
```

5. Redeploy in Coolify.

## Verify

1. Open `/admin/settings#api-manage-pane`.
2. Change API mode or update a token and confirm API Manage remains active.
3. Click **Check API endpoints** and verify the static routes pass.
4. Click one row's **Check** button and confirm only that row updates without a page reload.
5. Click **Use** and paste the copied instruction into a text editor.
6. Run one Sync Source operation and confirm compact detailed diagnostics are shown.
7. Verify `/api/v1/capabilities` reports version `0.2.3.1`.
